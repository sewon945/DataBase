논리적 모델링 이란?
ERD를 기준으로 개체들을 데이터베이스가 지원하는 데이터로 변환하는 과정
즉, 데이터들을 구조적으로 설계하는 과정(상세한 설계도)

식별자 선택, 정규화, 관계설정 등 진행

세가지의 모델링 중 가장 중요한 과정

식별자 선택 - 엔터티 내에서 유일한 인스턴스를 식별할 수 있는 속성. 
               - 각 엔터티는 무조건 식별자를 하나 가지고 있어야 함.
정규화 란 - 데이터가 중복되지 않도록 엔터티를 쪼개는 것
            - 즉 중복 데이터 등 이상현상 방지

물리적 모델링 이란?
논리적 모델링 결과를 가지고 실제 데이터베이스로 구축하는 것



개념적 모델링에서 책이라는 개체(Entity엔터티)라고 불렀는데 물리적 모델링에선 테이블(Table)이라고 부름
속성은 열(Column컬럼)이라고 부르고 인스턴스는 행(Row)이라고 불림

SQL의 실행 순서
5. SELECT          출력하고 싶은 컬럼만 작성하기
1. FROM           데이터를 가져올 테이블 입력
2. WHERE          원하는 행만 가져오도록 조건 지정(필터링)
3. GROUP BY      특정 컬럼을 기준으로 그룹화 
4. HAVING         그룹화 상태에서 데이터에 조건 지정
6. ORDER BY      특정 컬럼으로 정렬

SELSECT 사용 방법
2. SELSECT        조회 할 컬럼 명 1, 컬럼 명2, ... 컬럼 명N
1. FROM           조회 할 테이블 명

자료형
문자형 - CHAR(길이)          - 2000byte만큼의 고정길이 문자열 데이터 저장 (최소 1byte)
         - VARCHAR2(길이)   - 4000byte만큼의 가변길이 문자열 데이터 저장소 (최소 1byte)
숫자형 - NUMBER(p, s)       - P는 전체 자리 수
                                   - S는 소수점 이하 자리 수
                                   - 생략을 하게 되면 전체 크기인 38 자릿수까지 입력 가능
날짜형 - DATE                 - 세기, 연, 월, 일, 시, 분, 초 저장이 가능
         - TIMESTAMP         - 연도, 월, 일, 시, 분, 초 + 밀리 초까지 입력 가능



   < WHERE >
WHERE절은 SELECT문으로 데이터를 조회할 때 특정 조건을 기준으로 원하는 행을 출력하는데 사용

  WHERE절 사용방법
3.   SELECT
1.    FROM
2.  WHERE   조회할 행을 선별하기 위한 조건식 ;


 논리 연산자란? 개발자에게 논리 조건이란 참(TRUE)이냐 거짓(FALSE)이냐 이다.
- 논리 조건의 종류 : AND , OR 조건

 논리연산자 사용방법
3. SELECT
1. FROM
2. WHERE      조건1
   AND/OR    조건2
   AND/OR    조건3
